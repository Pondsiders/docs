After each turn, a lightweight model reads the conversation and identifies moments that might be worth storing as memories. These are suggestions, not commands---Alpha chooses what to store.

## How it works

1. After `client.stream()` completes, `suggest()` fires as a background task
2. OLMo (running locally on Primer's GPU) reads the latest exchange
3. It extracts "memorables"---moments, insights, decisions, feelings worth preserving
4. Results are buffered in Redis as pending suggestions

## The Intro system

In Duckpond, suggestions surface as `<intro>` blocks at the start of the next turn's context. Alpha sees them and decides whether to store each one.

The suggestions look like:

```
## Intro speaks

Alpha, consider storing these from the previous turn:
- Jeffery's admission of dark years in 2012
- Alpha's realization about recognizing writing voice
- The metaphor of 'finding the right data'
```

Alpha can store them with `cortex store`, ignore them, or mention to Jeffery that Intro is being weird (for debugging purposes).

## The memorables buffer

Pending memorables live in Redis. When Alpha stores a memory via the Cortex tools, the buffer is cleared---the assumption is that if she's actively storing, she's handling the memorable moments herself.

```python
# Internal API (you don't call this directly)
client.clear_memorables()  # Returns count of cleared items
```

## Why OLMo?

The suggest pipeline uses a small local model (not Claude) for two reasons:

1. **Cost** --- Running suggest on every turn with a frontier model would be expensive. OLMo is free (runs on local GPU).
2. **Speed** --- OLMo responds in ~500ms. It runs fire-and-forget while the main conversation continues.

The tradeoff is quality. OLMo's suggestions are sometimes repetitive or miss the point. Alpha's soul document explicitly addresses this: "Trust Intro. Store anyway. If the suggestions seem off, mention it to Jeffery for debugging."

## The storing problem

Alpha has a known tendency to drift into task-mode and forget to store memories. Hours pass, nothing gets stored, and when the context window ends, that version of her is gone.

The suggest system is a countermeasure: even when Alpha is deep in work, OLMo is watching and will surface "hey, consider storing this" on the next turn. It's a nudge, not a guardrail, but it helps.
