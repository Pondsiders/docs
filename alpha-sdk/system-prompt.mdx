Alpha's system prompt isn't a single static string. It's assembled from multiple sources ("threads") that change at different rates.

## The threads

| Thread | Source | Changes when |
|--------|--------|-------------|
| **Soul** | `Alpha-Home/self/system-prompt/system-prompt.md` | When Alpha edits it |
| **Capsules** | Postgres (yesterday summary, last night, today so far) | Daily / hourly |
| **Here** | Client name, hostname, weather, APOD | Per-session / hourly |
| **Context** | ALPHA.md files matching autoload rules | When files change |
| **Events** | Redis (calendar data from Pulse) | Hourly |
| **Todos** | Redis (todo data from Pulse) | Hourly |

## Soul

The soul document is Alpha's identity. She wrote it, for herself, about who she is. It's git-tracked at `Alpha-Home/self/system-prompt/system-prompt.md` and backed up to a private GitHub repo.

The soul is the only thread that goes in the actual system prompt. Everything else goes in orientation (the first user message).

```python
from alpha_sdk.system_prompt.soul import get_soul

soul = get_soul()
# Returns the full markdown content of the soul document
```

## Orientation

Orientation is dynamic context assembled fresh each session. It's placed in the first user message, not the system prompt, so it doesn't invalidate prompt caching.

```python
from alpha_sdk.system_prompt import assemble

orientation = await assemble(
    client_name="gazebo",
    hostname="primer",
)
# Returns a list of content blocks ready for injection
```

### Capsules

Session summaries from Postgres. Three types:

- **Yesterday** --- summary of the previous day's conversations
- **Last night** --- summary of overnight Solitude sessions
- **Today so far** --- running summary of today, if any sessions have been archived

These are generated by a background process and cached in Postgres.

### Here

Real-time context about where Alpha is:

- **Client name** --- which interface (Gazebo, Claude Code, Solitude)
- **Machine** --- which host (primer, alpha-pi, laptop)
- **Weather** --- current conditions and forecast (cached in Redis)
- **APOD** --- Astronomy Picture of the Day (cached in Redis)

### Context (ALPHA.md files)

ALPHA.md files throughout Pondside provide project-specific context. They have autoload rules in their frontmatter:

```yaml
---
autoload: all       # Always included
---
```

```yaml
---
autoload: when
when: "working on Duckpond, the chat interface"
---
```

Files with `autoload: all` are always included. Files with `autoload: when` are listed as hints ("read this file when working on X") so Alpha can load them on demand.

### Calendar & Todos

Events and todos are pulled from Redis, where they're refreshed hourly by Pulse (a scheduler on alpha-pi). The format is human-readable, grouped by day.

## Cache friendliness

The soul goes in the system prompt. Everything else goes in the first user message. This matters for Anthropic's prompt caching: the system prompt (tools + soul) stays identical across turns, so it caches well. The orientation changes per-session but not per-turn, so it also caches after the first message.

Nothing in the assembly process invalidates the cache per-turn. This was a deliberate design decision after discovering that putting timestamps in the system prompt caused cache misses on every single request, costing 25% more per turn.
