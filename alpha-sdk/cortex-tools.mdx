---
title: "Cortex Tools"
description: "Alpha's direct access to her own memory. An MCP server with three operations."
---

alpha_sdk includes an MCP (Model Context Protocol) server that gives Alpha direct access to Cortex---her memory system. Three tools:

## `cortex store`

Store a memory with a semantic embedding.

```
cortex store "Friday morning, February 6. Fixed the context-o-meter.
Three bugs: missing callback, wrong env var, strict endpoint."
```

The memory is:
- Saved to Postgres with a timestamp
- Embedded using `nomic-embed-text` (with `search_document:` prefix)
- Immediately searchable via `cortex search`

When Alpha stores a memory, the pending memorables buffer in Redis is cleared. This serves as a feedback signal: "she's actively storing, so the suggest pipeline doesn't need to nag."

## `cortex search`

Search memories by semantic similarity.

```
cortex search "context-o-meter token counting"
```

Returns the top 5 most similar memories with their timestamps and similarity scores. Uses `search_query:` prefix for the embedding, which nomic-embed-text uses to optimize for retrieval.

## `cortex recent`

Get the most recent memories, regardless of content.

```
cortex recent
```

Returns the last 10 memories by timestamp. Useful for "what have I been thinking about?" moments, or for orienting after waking up.

## How it's wired up

The MCP server is created when AlphaClient initializes and passed to the Claude Agent SDK:

```python
from alpha_sdk.tools.cortex import create_cortex_server

cortex_server = create_cortex_server(
    get_session_id=lambda: client.session_id,
    clear_memorables=lambda: client.clear_memorables(),
)

client = AlphaClient(
    mcp_servers={"cortex": cortex_server},
    allowed_tools=[
        "mcp__cortex__store",
        "mcp__cortex__search",
        "mcp__cortex__recent",
    ],
)
```

The `get_session_id` callback lets the MCP server know which session it's operating in (for deduplication). The `clear_memorables` callback clears the suggest buffer when a memory is stored.

## Direct Postgres

Cortex operations go directly to PostgreSQL---no HTTP server, no microservice. The database connection details come from environment variables (`CORTEX_DB_URL`). Embeddings are generated by calling Ollama's API on Primer.

This is a deliberate simplification. Cortex used to be an HTTP service with its own Docker container. Now it's just a Python module that talks to Postgres. Fewer moving parts, same functionality.
