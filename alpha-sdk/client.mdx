---
title: "AlphaClient"
description: "The main wrapper. Long-lived, session-aware, handles everything automatically."
---

AlphaClient is the single entry point for using alpha_sdk. It wraps the Claude Agent SDK and adds soul injection, memory, orientation, archiving, and observability.

## Lifecycle

The SDK client is expensive to create (~4 seconds of warmup). AlphaClient handles this by keeping one client alive and reusing it across conversations.

```python
from alpha_sdk import AlphaClient

# Create once at application startup
client = AlphaClient(
    cwd="/Pondside",
    client_name="my-app",
    allowed_tools=["Read", "Write", "Bash", "Grep", "Glob"],
    mcp_servers={"cortex": cortex_server},
)

# Connect (starts proxy, initializes SDK)
await client.connect(session_id="abc123")  # Resume existing session
# or
await client.connect()  # New session

# Use for conversation
await client.query("Hello!")
async for event in client.stream():
    handle(event)

# Clean up
await client.disconnect()
```

## Constructor parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `cwd` | `str` | `"/Pondside"` | Working directory for the agent |
| `client_name` | `str` | `"alpha_sdk"` | Name of the client (for logging, HUD display) |
| `hostname` | `str \| None` | auto-detected | Machine hostname |
| `allowed_tools` | `list[str] \| None` | `None` | List of tool names the agent can use |
| `mcp_servers` | `dict \| None` | `None` | MCP server configurations |
| `archive` | `bool` | `True` | Whether to archive turns to Postgres |
| `permission_mode` | `str` | `"default"` | How to handle tool permissions (`"default"`, `"bypassPermissions"`) |
| `on_token_count` | `callback \| None` | `None` | Called when token count changes: `(count, window) -> None` |

## Session handling

AlphaClient manages session switching automatically:

```python
# Resume an existing session
await client.query(prompt, session_id="abc123")

# Same session - reuses existing SDK client
await client.query(another_prompt, session_id="abc123")

# Different session - closes and recreates SDK client
await client.query(prompt, session_id="xyz789")

# New session - creates fresh, captures assigned ID
await client.query(prompt)
print(client.session_id)  # The ID assigned by the SDK
```

## Session discovery

```python
# List available sessions (static method)
sessions = await AlphaClient.list_sessions(cwd="/Pondside")
# Returns: [SessionInfo(id="30bb8d6f...", created=..., preview="Hello..."), ...]

# Get filesystem path for a session
path = AlphaClient.get_session_path("30bb8d6f...", cwd="/Pondside")
```

## Context manager

For one-shot use:

```python
async with AlphaClient(cwd="/Pondside") as client:
    await client.query(prompt, session_id=session_id)
    async for event in client.stream():
        yield event
```

The context manager handles `connect()` and `disconnect()` automatically.

## Token counting

Token counting happens automatically via CompactProxy. To receive updates:

```python
# Option 1: Set at construction
client = AlphaClient(
    on_token_count=lambda count, window: print(f"{count}/{window}")
)

# Option 2: Swap per-turn (useful for SSE streams)
client.set_token_count_callback(my_callback)
```

The callback fires whenever the token count increases. The `max()` filter ensures warmup requests don't produce noise---only the largest request per session triggers the callback.

## Properties

| Property | Type | Description |
|----------|------|-------------|
| `session_id` | `str \| None` | Current session ID |
| `connected` | `bool` | Whether the SDK client is active |
| `token_count` | `int` | Current input token count |
| `context_window` | `int` | Context window size (default 200,000) |

## What happens automatically

When you call `client.query(prompt)`:

1. **Recall** --- Cortex is searched for memories relevant to the prompt
2. **Orientation** --- If this is the first turn, dynamic context is assembled
3. **Injection** --- Memories and orientation are added to the user message
4. **Query** --- The prompt is sent to the Claude Agent SDK
5. **Archiving** --- After the turn completes, it's archived to Postgres (fire-and-forget)
6. **Suggest** --- A lightweight model identifies memorable moments (fire-and-forget)

You don't call any of these explicitly. They just happen.
