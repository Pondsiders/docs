## The big picture

```
alpha_sdk/
├── __init__.py              # Exports AlphaClient
├── client.py                # AlphaClient - the main wrapper
├── compact_proxy.py         # Localhost proxy for compact rewriting + token counting
├── system_prompt/
│   ├── __init__.py          # assemble() - builds the full system prompt
│   ├── soul.py              # The soul doc (from git-tracked markdown)
│   ├── capsules.py          # Yesterday, last night, today (from Postgres)
│   ├── here.py              # Client, hostname, weather, astronomy
│   ├── context.py           # ALPHA.md files (autoload rules)
│   ├── calendar.py          # Events (from Redis)
│   └── todos.py             # Todos (from Redis)
├── memories/
│   ├── cortex.py            # store, search, recent (direct Postgres)
│   ├── recall.py            # Smart recall (embedding + OLMo query extraction)
│   └── suggest.py           # OLMo memorables extraction
├── tools/
│   └── cortex.py            # MCP server for memory tools
├── archive.py               # Turn archiving to Postgres
├── sessions.py              # Session discovery and management
└── observability.py         # Logfire setup
```

## What it replaced

alpha_sdk unified several standalone services into one importable library:

| Before | After |
|--------|-------|
| **Deliverator** (proxy service on alpha-pi) | Gone---no headers to promote |
| **The Loom** (proxy service on alpha-pi) | Logic moved into compact_proxy.py |
| **Argonath** (observability service on alpha-pi) | Logic moved into observability.py |
| **Duckpond's memories/** | Moved to alpha_sdk/memories/ |
| **Duckpond's context building** | Moved to alpha_sdk/system_prompt/ |
| **Cortex HTTP server** | Replaced by direct Postgres access |
| **The metadata envelope/canary system** | Gone---we control the request directly |

## What remains external

- **Postgres** --- memories, archival, capsule summaries. Runs on alpha-pi.
- **Redis** --- caching (weather, calendar, todos, memorables buffer). Runs on alpha-pi.
- **OLMo** --- recall query extraction and memorables suggestion. Runs on Primer (GPU).
- **Pulse** --- schedules Routines (Solitude). Runs on alpha-pi.

## The proxy pattern

Claude Agent SDK sends all API requests to `ANTHROPIC_BASE_URL`. alpha_sdk sets this to `http://localhost:{random_port}` and runs a minimal HTTP proxy (CompactProxy) that:

1. Receives the request from the SDK
2. Checks for auto-compact signatures and rewrites them if found
3. Fires a token-counting request (async, non-blocking)
4. Forwards to `https://api.anthropic.com`
5. Streams the response back

The proxy starts automatically when AlphaClient connects and stops when it disconnects. No configuration needed.

## Request flow

A single user message triggers this sequence:

```
User sends message
    │
    ├── recall() searches Cortex for relevant memories
    │   ├── Embedding search (pgvector cosine similarity)
    │   └── OLMo extracts search queries from the prompt
    │
    ├── Memories injected into user message content blocks
    │
    ├── query() sends to Claude Agent SDK
    │   └── SDK sends to CompactProxy
    │       ├── Compact detection & rewriting (if needed)
    │       ├── Token counting (fire-and-forget)
    │       └── Forward to Anthropic API
    │
    ├── stream() yields response events
    │
    ├── suggest() runs (fire-and-forget)
    │   └── OLMo identifies memorable moments
    │
    └── archive() saves turn to Postgres
```

## Session storage

The Claude Agent SDK stores sessions as append-only JSONL files:

```
~/.claude/projects/{formatted_cwd}/{session_id}.jsonl
```

Where `formatted_cwd` is the realpath with `/` replaced by `-`. For `/Pondside`, that's `-Pondside`.

AlphaClient's `list_sessions()` and `get_session_path()` encapsulate all path formatting. Consumers never need to know about JSONL files.
