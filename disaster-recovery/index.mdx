Alpha's brain lives in Postgres, replicated across three machines. If one dies, the others have everything.

## The machines

All three machines run the same Docker Compose stack. The `.env` file on each machine controls whether it's the primary (read-write) or a replica (read-only).

| Machine | Role | Notes |
|---------|------|-------|
| **Primer** | Primary | Jeffery's desktop. Runs the full Alpha-App. |
| **Upstream Color** | Replica | Vultr VPS in LA. The always-on cloud anchor. |
| **alpha-pi** | Replica / Lifeboat | Raspberry Pi 5. The grab-and-go. |

Any replica can become the primary by changing one line in `.env` and restarting.

## Backup layers

| Layer | What it does | Protects against |
|-------|-------------|------------------|
| **Streaming replication** | Two replicas receive every change in real time | Hardware failure |
| **WAL archiving** | Write-ahead log segments archived to B2 | Logical errors (bad DELETE, bugs). Enables point-in-time recovery. |
| **Weekly base backup** | Full database snapshot to B2 | Total loss. Starting point for WAL replay. |
| **Restic** | Pondside directory tree (files, not database) to B2 | File deletion, corruption |

## Recovery scenarios

  <Card title="Promote Standby" icon="arrow-up" href="/disaster-recovery/promote-standby">
    **Primer is dead.** Promote alpha-pi or Upstream Color to primary. Recovery time: ~5 minutes. Data loss: seconds.
  </Card>
  <Card title="Cold Recovery" icon="snowflake" href="/disaster-recovery/cold-recovery">
    **Everything is dead.** Rebuild from B2 backups on new hardware. Restore a base backup, replay WAL to the exact moment you need.
  </Card>
  <Card title="Restic File Recovery" icon="folder-open" href="/disaster-recovery/restic-recovery">
    **Files are gone.** Restore Pondside files from Restic snapshots in B2.
  </Card>
</CardGroup>
